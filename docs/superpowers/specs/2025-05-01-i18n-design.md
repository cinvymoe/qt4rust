# i18n 多语言支持设计文档

**项目**: qt4rust (Qt + Rust 起重机监控系统)  
**版本**: 1.0  
**日期**: 2025-05-01  
**状态**: 已批准

---

## 1. 目标

为 qt4rust 应用添加**多语言支持**，实现：

- 运行时动态切换语言（无需重启）
- 中文 + 英文（预留扩展接口）
- 解耦的翻译后端架构
- 语言偏好持久化到本地配置文件

---

## 2. 架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        QML UI Layer                             │
│                                                                  │
│  text: Tr.t("monitoring.currentLoad")                          │
│  text: Tr.t("alarm.warning")                                    │
│  text: Tr.t("settings.language")                                │
├──────────────────────────────────────────────────────���──────────┤
│              TranslationBridge (cxx-qt)                        │
│         暴露 tr(key) -> QString 给 QML                          │
│         signal: languageChanged()                              │
├─────────────────────────────────────────────────────────────────┤
│                   TranslationProvider                          │
│     trait Translate { fn t(&self, key: &str) -> String; }      │
├─────────────────────┬─────────────────────┬────────────────────┤
│  FluentBackend      │   QtQmBackend       │   (Future...)      │
│  (.ftl files)       │   (.qm files)       │                    │
│  - 复数支持          │   - Qt原生          │                    │
│  - 变量插值          │   - Qt Linguist     │                    │
│  - 零分配性能        │                     │                    │
└─────────────────────┴─────────────────────┴────────────────────┘
```

### 2.2 核心原则

1. **Trait 抽象**：后端可插拔，便于测试和扩展
2. **运行时热切换**：语言切换立即生效，无需重启
3. **配置持久化**：语言偏好保存到本地文件
4. **优雅降级**：翻译缺失时回退到 key 本身

## 3. 组件设计

### 3.1 Trait 定义 (`traits.rs`)

```rust
pub trait Translate {
    /// 根据 key 获取翻译
    fn t(&self, key: &str) -> String;
    
    /// 根据 key 获取翻译，支持变量插值
    fn t_with_args(&self, key: &str, args: &[(&str, &str)]) -> String;
    
    /// 获取当前语言代码
    fn current_locale(&self) -> &str;
}
```

### 3.2 Fluent Backend (`fluent_backend.rs`)

- 使用 `fluent-bundle` 加载 `.ftl` 文件
- 支持复数形式、变量插值
- 编译时校验 key 存在（通过测试）

### 3.3 Locale Manager (`locale_manager.rs`)

职责：
- 加载/保存语言偏好到配置文件
- 检测系统默认语言
- 提供语言切换 API

配置路径：`config/app_config.toml`

```toml
[app]
language = "zh-CN"
```

## 5. QML 集成

### 5.1 导入翻译模块

```qml
import "i18n"  // 导入 TranslationBridge
```

### 5.2 使用翻译

```qml
Text {
    text: Tr.t("monitoring.currentLoad")
}

Text {
    text: Tr.t("alarm.warningMessage", { percentage: "90%" })
}
```

### 5.3 语言切换 UI

```qml
ComboBox {
    model: [
        { locale: "zh-CN", name: "简体中文" },
        { locale: "en-US", name: "English" }
    ]
    onCurrentIndexChanged: {
        Tr.setLocale(model[currentIndex].locale)
    }
}
```

## 7. 错误处理策略

| 场景 | 处理方式 |
|------|----------|
| Key 不存在 | 返回 key 本身 + warning log |
| 语言文件加载失败 | 回退到 zh-CN |
| 配置文件损坏 | 使用默认值，重建配置 |
| 变量插值缺失参数 | 忽略该变量，显示原始文本 |
